The Single File Agent

Documentation + Contract + Code = One File

Every agent defines its metadata using native code ‚Äî Python dicts or TypeScript objects. No YAML, no config files, no magic. Just code. Read the manifesto ‚Üí

"""
MyAgent ‚Äî What this agent does.
"""
import json
from openrappter.agents.basic_agent import BasicAgent

class MyAgent(BasicAgent):
    def __init__(self):
        self.name = 'MyAgent'
        self.metadata = {
            "name": self.name,
            "description": "What this agent does",
            "parameters": {
                "type": "object",
                "properties": {
                    "query": {"type": "string", "description": "User input"}
                },
                "required": []
            }
        }
        super().__init__(name=self.name, metadata=self.metadata)

    def perform(self, **kwargs):
        query = kwargs.get('query', '')
        # self.context ‚Üí enriched by data sloshing
        return json.dumps({"status": "success", "result": "..."})
import { BasicAgent } from './BasicAgent.js';
import type { AgentMetadata } from './types.js';

export class MyAgent extends BasicAgent {
  constructor() {
    const metadata: AgentMetadata = {
      name: 'MyAgent',
      description: 'What this agent does',
      parameters: {
        type: 'object',
        properties: {
          query: { type: 'string', description: 'User input' }
        },
        required: []
      }
    };
    super('MyAgent', metadata);
  }

  async perform(kwargs: Record<string, unknown>): Promise<string> {
    const query = kwargs.query as string;
    // this.context ‚Üí enriched by data sloshing
    return JSON.stringify({ status: 'success', result: '...' });
  }
}

Data Slush

Agent-to-agent signal pipeline

Agents return a data_slush field ‚Äî curated signals from live results ‚Äî that the framework extracts and feeds into the next agent's context. No LLM needed between calls.

# Agent A ‚Äî return data_slush with curated signals
def perform(self, **kwargs):
    weather = fetch_weather(kwargs.get('query'))
    return json.dumps({
        "status": "success",
        "result": weather,
        "data_slush": {
            "source_agent": self.name,
            "temp_f": 65,
            "condition": "cloudy",
        }
    })

# Chain: A's data_slush feeds into B's context
result_a = agent_a.execute(query="Smyrna GA")
result_b = agent_b.execute(
    query="...",
    upstream_slush=agent_a.last_data_slush
)
# B's self.context['upstream_slush'] has A's signals
// Agent A ‚Äî return data_slush with curated signals
async perform(kwargs: Record<string, unknown>): Promise<string> {
  const weather = await fetchWeather(kwargs.query);
  return JSON.stringify({
    status: 'success',
    result: weather,
    data_slush: {
      source_agent: this.name,
      temp_f: 65,
      condition: 'cloudy',
    }
  });
}

// Chain: A's data_slush feeds into B's context
const resultA = await agentA.execute({ query: 'Smyrna GA' });
const resultB = await agentB.execute({
  query: '...',
  upstream_slush: agentA.lastDataSlush,
});
// B's this.context.upstream_slush has A's signals

How Copilot Fits In

Cloud AI, local data

GitHub Copilot provides the AI inference layer. Your agent data ‚Äî memory, config, state ‚Äî never leaves your machine.

üß†

AI Inference Layer

Local Data
